# q1 问题修复完成说明

对应需求见 [q1.md](q1.md)。本轮已在 Agent 模式下落盘代码，`npm run build` 通过。

## 1. 画布滚轮放大

- 在 [`src/statusbar.ts`](../../src/statusbar.ts) 的 `initZoom` 中，对 `#canvas-outer` 监听 `wheel`，`preventDefault` 且 `passive: false`。
- 与底部 `+/−/适应` 共用 `patchViewport` + `applyZoom()`。

## 2. 元素拖拽

- **根因**：`mousedown` 后 `store.set({ selectedId })` 触发全量 `renderCanvas()`，`innerHTML` 清空导致正在拖拽的节点被卸载。
- **修复**：[`src/canvas.ts`](../../src/canvas.ts) 改为**增量** `renderCanvas`：按 `data-id` 复用已有 DOM，仅更新布局、内容签名与选中态。
- **缩放**：用 `DOMMatrixReadOnly` + `matrixTransform(m.inverse())` 将指针换算到画布本地坐标（兼容 `transform: scale`）。

## 3. 单击选中

- 元素上 `mousedown` 增加 `stopPropagation`，避免与外侧逻辑冲突；增量渲染后单击即可稳定选中。

## 4. 属性输入单次一字

- **根因**：`input` → `updateElement` → 全量 `renderProperties` 重建表单导致失焦。
- **修复**：[`src/properties.ts`](../../src/properties.ts) 中文本与透明度滑块使用 `updateElementNoNotify` + 仅 `renderCanvas()` + `scheduleSave()`（见 [`src/store.ts`](../../src/store.ts)）。

## 5. 坐标轴 / 元素样式

- 轴线：`renderAxisLabels` 向 `#canvas` 写入 `--axis-x-line`、`--axis-y-line` 及箭头用 `--axis-x-arrow`、`--axis-y-arrow`；[`src/style.css`](../../src/style.css) 中轴线与箭头改用上述变量。
- 侧栏改轴颜色后，画布轴线颜色会随状态刷新。
- 元素颜色/图标仍由 `fillElementContent` 与属性面板驱动。

## 6. Lucide 图标库

- 依赖：`lucide`（已列入 `package.json`）。
- [`src/icons.ts`](../../src/icons.ts)：预设 `PRESET_ICON_KEYS` + `getLucideSvg`。
- [`src/types.ts`](../../src/types.ts)：`QuadElement.iconName`；[`src/storage.ts`](../../src/storage.ts)：`normalizeAppState` 为缺省项补 `star`。
- 画布卡片/标签/图标/文字均以 SVG 呈现；属性面板为图标网格；图层面板列表用小号 Lucide。
