# Q2 问题修复记录

## 1. 单击元素可编辑文字

- **select** 工具下，单击元素（位移 < 3px 视为点击而非拖拽）后弹出 `inline-edit-input` 覆盖层。
- 按 Enter 或失焦提交，按 Escape 取消；`icon` 类型无文案，不弹出编辑框。
- 事件 `stopPropagation` 防止点击穿透到画布导致取消选中。

## 2. 消除全场抖动

- 删除 `renderCanvas()` 末尾按 z 序对所有节点 `container.appendChild(dom)` 的重排循环，改由 `style.zIndex` 控制叠放。
- `onMouseUp` 新增 `didDrag` 标记与 `DRAG_THRESHOLD`，仅在实际拖拽且位置变化时才 `store.updateElement()`，避免无意义 `notify` → 全量刷新。

## 3. 模板切换时同步四象限

- `QuadrantConfig` 增加可选 `tint?: string` 字段，存储象限背景 RGBA 值。
- `initTemplates` 改为声明式 `TEMPLATES` 映射，每个模板同时设置 `axes` 和完整 `quadrants`（名称、emoji、tint）。
- `renderAxisLabels()` 中将 `quadrants[i].tint` 注入 `#canvas` 的 `--q1` … `--q4` CSS 变量，驱动 `.quad` 背景色联动。

## 4. 工具栏可用性

- 非 select 工具时，**单击空白画布** 即可创建元素（与双击逻辑复用 `createElementAtPoint`）。
- `line` 工具默认图标映射为 `target`（占位），后续可扩展为真正的矢量连线。
